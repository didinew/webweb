## 一句话定义（先背）

订单簿深度 = 在不同价格档位上，市场能承接多少买卖量

说人话就是：

你这笔单子下去，市场能不能“接得住”

---

## 订单簿长什么样？

订单簿永远分两边：
- 卖单（Asks）：价格由低到高
- 买单（Bids）：价格由高到低

#### 示例
```
卖：
30100  0.10
30110  0.20
30120  0.50

买：
30090  0.15
30080  0.30
30070  0.60
```


---

## 什么叫「深」？什么叫「浅」？

### 深度深（流动性好）
- 每一档量都很大
- 大单不容易推价格

```
30100  5 BTC
30110  8 BTC
```

### 深度浅（流动性差）
- 每档只有一点点
- 稍微一吃就飞价

```
30100  0.01 BTC
30110  0.02 BTC
```

## 深度和「部分成交」的关系（串起来）

- 你下：市价买 1 BTC
- 深度浅：只能成交 0.3 BTC → 部分成交 → 滑点巨大
- 深度深：一次性成交 1 BTC → 成交价稳定

> 深度 = 部分成交 + 滑点的根源

## 为什么交易所要展示「深度图」？

深度图本质就是：

把订单簿“累计”画出来

#### 含义
- 横轴：价格
- 纵轴：累计数量

#### 能快速看出
- 哪边墙厚
- 哪边容易被打穿
五、工程师视角：深度是怎么来的？

数据来源： 撮合引擎内存中的订单簿
常见获取方式：

	•	REST：fetchOrderBook
	•	WS：实时增量更新

六、ccxt 看订单簿深度（你能立刻用）
const orderBook = await exchange.fetchOrderBook('BTC/USDT', 10)

console.log(orderBook.bids)
/*
[
  [30090, 0.15],
  [30080, 0.30],
]
*/

console.log(orderBook.asks)
/*
[
  [30100, 0.10],
  [30110, 0.20],
]
*/
📌 每一项含义：
[价格, 数量]

七、怎么用深度做「下单决策」？

1️⃣ 判断会不会滑点
我这单的数量 > 前 N 档深度之和？

2️⃣ 判断是否会部分成交
市价单 + 深度浅 = 高概率部分成交

3️⃣ 策略级用法
	•	拆单
	•	冰山单
	•	TWAP / VWAP

八、CEX vs DEX 深度对比（理解很重要）

| 维度 | CEX | DEX（AMM） |
| --- | --- | --- |
| 深度来源 | 订单 | 资金池 |
| 是否真实挂单 | 是 | 否 |
| 深度形态 | 阶梯状 | 曲线 |
| 吃深度结果 | 滑点小 | 滑点大 |

你的下单量 ≈ 你要吃掉的深度
