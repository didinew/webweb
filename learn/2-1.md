Day 1｜Web3 基础 & 账户模型（认知纠偏）

核心一句话：
Web3 不是“登录方式变了”，而是 身份所有权从平台转移到了私钥。

⸻

一、Web2 vs Web3（不是技术差异，是“权力结构”）

Web2（传统互联网）
	•	身份：username + password
	•	所有者：平台
	•	你只是“被授权使用账号”

你 → 平台 → 数据 / 资产

Web3（去中心化）
	•	身份：私钥
	•	所有者：你自己
	•	钱包 = 身份 + 资产 + 授权中心

你(私钥) → 链 → 资产 / 合约

👉 认知纠偏点
❌ 钱包不是“第三方登录工具”
✅ 钱包是 身份本身

二、区块链账户模型（面试必考）

1️⃣ EOA（Externally Owned Account）

外部账户（普通钱包）

特点：
	•	由 私钥控制
	•	可以：
	•	转账
	•	调用合约
	•	签名

MetaMask / OKX Wallet / Ledger

2️⃣ Contract Account（合约账户）

特点：
	•	没有私钥
	•	只能被调用
	•	行为由代码定义

ERC20 / DEX / NFT 合约

👉 面试加分点

区块链世界只有 EOA 能“发起行为”，

三、地址 / 公钥 / 私钥（90%的人说不清）

正确关系（非常重要）

私钥 → 公钥 → 地址

	•	私钥：256 位随机数（本质）
	•	公钥：椭圆曲线计算结果
	•	地址：公钥 hash 后的后 20 字节

常见误区纠正

❌ 地址 = 账号
❌ 钱包里存的是资产
❌ 钱包帮你“保管钱”

✅ 资产在链上
✅ 钱包只做三件事：
	1.	管私钥
	2.	生成签名
	3.	发交易


⸻

四、为什么“私钥 = 身份”

想象一个极端例子
	•	没有邮箱
	•	没有手机号
	•	没有用户名

你如何证明你是你？

👉 用私钥签名
我能用这个私钥签名
→ 我就是这个地址的拥有者

区块链只认一件事：

能否验证签名

五、最小代码验证（Node.js）

不是“会写”，而是写了就懂

import { Wallet } from 'ethers'

// 创建随机钱包
const wallet = Wallet.createRandom()

console.log('address:', wallet.address)
console.log('privateKey:', wallet.privateKey)
console.log('mnemonic:', wallet.mnemonic.phrase)

你现在应该理解：
	•	地址是“算出来的”
	•	钱包可以无限生成
	•	没有注册流程

六、Day 1 面试必杀回答（直接背）

Q：Web3 的账户和 Web2 有什么本质区别？

标准回答：

Web2 的账户是平台分配的身份，
Web3 的账户是私钥衍生的身份。
你是否是账户所有者，只取决于你是否掌握私钥。

⸻

Q：合约账户为什么不能主动发交易？

标准回答：

因为它没有私钥，
所有执行必须由 EOA 触发。

⸻

Q：钱包到底是什么？

标准回答：

钱包不是账号，也不存资产，
它只是一个 私钥管理和签名工具。

⸻

七、今天你必须真正想通的 3 件事

1️⃣ 钱包 ≠ 登录方式
2️⃣ 私钥丢失 = 身份永远丢失
3️⃣ 合约永远不能“主动”发交易
